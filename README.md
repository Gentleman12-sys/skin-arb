# СПРЕД — арбитраж скинов CS2 · Steam ↔ LIS-Skins / CS.money

Программа сверки цен для **выгодного вывода денег со скинов CS2**. Ты вводишь
сумму, которую хочешь получить на карту, а СПРЕД находит скин, который в Steam
стоит дешевле всего относительно того, что за него платит площадка скупки —
**LIS-Skins** или **CS.money** (берётся та, что платит больше).

> **Спред ≠ чистый профит.** Спред — это разница цен двух площадок на момент
> запуска. Реальная выгода меньше: комиссия Steam ~15%, курс валют и удержание
> выплат на стороне площадки. Цифры — ориентир, а не гарантия.

---

## Идея

Когда выводишь деньги через площадки вроде LIS-Skins, выгодность зависит от
конкретного скина: за одну и ту же выплату один скин в Steam стоит дороже,
другой — дешевле.

| | Steam | Площадка платит | Спред |
|---|---:|---:|---:|
| AK-47 (вариант A) | $70.0 | $80.0 | +$10.0 |
| **AK-47 (вариант B)** | **$66.2** | **$80.0** | **+$13.8 ← выгоднее** |

Оба варианта дают одну выплату ($80), но второй дешевле в Steam — значит
выгоднее. Программа сама находит такие варианты и ставит их наверх.

**Что нового по сравнению с исходной постановкой:** добавлена вторая площадка
скупки — **CS.money**. В режиме «Лучшая» по каждому скину сравниваются выплаты
LIS и CS.money, и берётся максимальная.

---

## Почему нужен бэкенд (а не просто один HTML)

Идеально было бы уместить всё в один `.html`, но цены из браузера напрямую не
взять:

- **Steam** (`priceoverview`) и **LIS / CS.money** не отдают ответы в браузер —
  блокирует **CORS**. У Steam вдобавок жёсткий лимит (~20 запросов/мин → ошибка 429).
- Нужны кэш на диск, паузы между запросами Steam и хранение токенов — это
  серверная работа.

Поэтому проект устроен так:

- **`spread-arbitrage.html`** — интерфейс. Открывается сам по себе и **работает
  в демо-режиме** на встроенных мок-данных (никакого сервера не надо).
- **`server.js`** — бэкенд на **чистом Node.js без единой зависимости** (только
  встроенные модули, `npm install` не требуется). Он проксирует и кэширует
  LIS / CS.money / Steam / курс валют, считает спред и отдаёт готовый результат.

Когда сервер запущен, страница **сама его находит** и переключается в
**live-режим** с реальными ценами (бейдж в шапке меняется `demo → live`). Если
сервера нет — остаётся демо.

## Быстрый старт

### Вариант 1. Демо (без сервера)
Просто открой `spread-arbitrage.html` в браузере. Всё работает на демо-данных —
видно интерфейс, расчёт спреда, сортировку, режим «чистыми», выбор площадки и валюты.

### Вариант 2. Live (реальные цены)
Нужен только Node.js (≥16), зависимостей нет.

```bash
node server.js            # поднимет сервер на http://localhost:8787
# открой http://localhost:8787 в браузере
```

Без настройки источников сервер всё равно отдаёт демо-данные (и честно пишет об
этом предупреждением). Чтобы пошли **реальные** цены — подключи источники ниже.

## Подключение реальных цен

Самый надёжный путь (как в исходном ТЗ) — **локальный JSON-файл прайс-листа**,
скачанный из браузера в залогиненном аккаунте. Парсер универсальный и понимает
разные форматы (массив объектов, словарь `{имя: цена}`, обёртки `items/data`,
цены строкой или числом).

```bash
# LIS из локального файла + CS.money из локального файла
LIS_PRICES_FILE=./data/lis_prices.json \
CSMONEY_PRICES_FILE=./data/csmoney_prices.json \
MODE=live node server.js
```

Цены во всех источниках считаются **в USD** (LIS/CS.money отдают доллары),
пользователю показываются в выбранной валюте по курсу USD→валюта.

### Все переменные окружения

| Переменная | Назначение | По умолчанию |
|---|---|---|
| `PORT` | порт сервера | `8787` |
| `MODE` | `demo` / `live` / `auto` | `auto` |
| `LIS_PRICES_FILE` | путь к локальному прайсу LIS (приоритет №1) | — |
| `LIS_API_TOKEN` | токен LIS API (приоритет №2) | — |
| `LIS_API_BASE` | база LIS API | `https://api.lis-skins.com/v1` |
| `LIS_BULK_URL` | bulk-JSON по прямой ссылке (приоритет №3) | публичный bulk |
| `CSMONEY_PRICES_FILE` | путь к локальному прайсу CS.money | — |
| `CSMONEY_URL` | JSON-эндпоинт прайса CS.money | — |
| `FX_PROVIDER_URL` | источник курсов (USD-база, без ключа) | `open.er-api.com` |
| `STEAM_DELAY_MS` | пауза между запросами Steam | `3500` (~17/мин) |
| `STEAM_MAX_LOOKUPS` | потолок «свежих» запросов Steam на один поиск | `40` |
| `STEAM_TTL` | TTL кэша цен Steam, сек | `21600` (6 ч) |
| `PRICES_TTL` | TTL кэша прайс-листов, сек | `1800` (30 мин) |
| `STEAM_FEE` | комиссия Steam для режима «чистыми» | `0.15` |
| `CACHE_DIR` | папка дискового кэша | `.cache` |

> Прямые API LIS/CS.money и Steam могут блокировать запросы (403/429) или менять
> формат — это нормально. Поэтому локальный файл прайса надёжнее, а сервер при
> недоступности источника аккуратно откатывается на демо и пишет предупреждение.

## Как это работает (по шагам)

1. **Прайсы целиком.** Один запрос на каждую площадку — вся таблица цен LIS и
   CS.money в USD. Результат кэшируется на диск (ни LIS, ни Steam не умеют искать
   «дай все скины по цене ~8000₽» — поиск только по названию, поэтому фильтруем
   по цене локально).
2. **Лучшая выплата + коридор.** По каждому скину берётся, кто платит больше
   (LIS или CS.money). Вокруг суммы строится ценовой коридор ±5% — в него
   попадают кандидаты.
3. **Сверка со Steam.** Только для кандидатов запрашивается цена Steam Market
   (минимальная, иначе медианная), в USD. С паузами, кэшем и повторами по 429.
4. **Спред и рейтинг.** Спред = выплата − цена Steam (в режиме «чистыми» ещё
   минус ~15%). Кандидаты сортируются по спреду; сверху — лучшая сделка со
   ссылками на обе площадки. Скины, по которым цена в Steam не нашлась
   (несовпадение `market_hash_name` — ★, StatTrak™), выносятся в отдельный список
   «не вошли в рейтинг».

## CLI

`server.js` — это ещё и консольная утилита.

```bash
node server.js search 8000                      # поиск по сумме (рубли, ±5%, лучшая площадка)
node server.js search 8000 --currency KZT --corridor 8 --exchange csmoney --net
node server.js search 100  --currency USD --exchange lis
node server.js fx                               # текущие курсы валют
node server.js steam "AK-47 | Asiimov (Field-Tested)"   # цена одного скина в Steam
node server.js diag --exchange lis              # проверить формат прайс-листа
node server.js serve --port 9000                # поднять сервер на другом порту
```

Флаги `search`: `--currency RUB|USD|EUR|KZT|UAH`, `--corridor <%>`,
`--exchange best|lis|csmoney`, `--net` (минус комиссия Steam), `--mode auto|live|demo`.

Пример вывода (демо):

```
  Режим: demo · площадка: best · валюта: RUB
  Цель: 8 000 RUB (~$86.96) · коридор 7 600–8 400 RUB · кандидатов: 10

  #  Скин                                   Площ.        платит     Steam     спред      %
```

## HTTP API

Все ответы — JSON, с заголовком `Access-Control-Allow-Origin: *` (можно дёргать
даже из `file://`).

| Endpoint | Описание |
|---|---|
| `GET /api/health` | статус, режим, настроенные источники |
| `GET /api/search?amount=&currency=&corridor=&exchange=&net=&mode=` | главный расчёт: возвращает `best`, `rows[]`, `unmatched[]`, `fx`, `meta`, `warnings` |
| `GET /api/fx` | курсы валют (USD-база) |
| `GET /api/steam?name=<market_hash_name>` | цена одного скина в Steam |
| `GET /api/prices?exchange=lis\|csmoney&limit=N` | срез прайс-листа площадки (диагностика) |

Поля строки результата (`rows[i]`): `name`, `wear`, `mhn` (market_hash_name),
`exchange` (`lis`/`csmoney`), `lis`, `csmoney`, `payout`, `steam`, `spread`,
`pct`, `links`. Все денежные значения — в **USD**; фронт переводит их в выбранную
валюту по `fx.rate`.

## Ограничения и нюансы

- **Лимиты Steam.** Эндпоинт цен ограничен (~20 запросов/мин). Поэтому
  проверяются только кандидаты из коридора, между запросами пауза, результаты
  кэшируются, при 429 — повтор с увеличенной паузой. Если кандидатов больше
  лимита за один заход — остаток помечается «дозапросим», достаточно повторить
  поиск (кэш дозаполнится).
- **Несовпадение имён.** Имя на площадке должно совпадать с `market_hash_name` в
  Steam, включая ★ и StatTrak™. Несовпавшие показываются отдельным списком, а не
  отбрасываются молча.
- **Точность.** Цены на всех площадках меняются — результат актуален на момент запуска.
