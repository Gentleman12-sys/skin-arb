# СПРЕД — арбитраж скинов CS2 · Steam ↔ LIS-Skins

Программа сверки цен для **выгодного вывода денег со скинов CS2**. Ты вводишь
сумму, которую хочешь получить на карту, а СПРЕД находит скин, который в Steam
стоит дешевле всего относительно того, что за него платит **LIS-Skins**.

> **Спред ≠ чистый профит.** Спред — это разница цен двух площадок на момент
> запуска. Реальная выгода меньше: комиссия Steam ~15%, курс валют и удержание
> выплат на стороне LIS. Цифры — ориентир, а не гарантия.

---

## Идея

Когда выводишь деньги через площадки вроде LIS-Skins, выгодность зависит от
конкретного скина: за одну и ту же выплату один скин в Steam стоит дороже,
другой — дешевле.

| | Steam | LIS платит | Спред |
|---|---:|---:|---:|
| AK-47 (вариант A) | $70.0 | $80.0 | +$10.0 |
| **AK-47 (вариант B)** | **$66.2** | **$80.0** | **+$13.8 ← выгоднее** |

Оба варианта дают одну выплату ($80), но второй дешевле в Steam — значит
выгоднее. Программа сама находит такие варианты и ставит их наверх.

---

## Почему нужен бэкенд (а не просто один HTML)

Идеально было бы уместить всё в один `.html`, но цены из браузера напрямую не
взять:

- **Steam** (`priceoverview`) и **LIS-Skins** не отдают ответы в браузер —
  блокирует **CORS**. У Steam вдобавок жёсткий лимит (~20 запросов/мин → ошибка 429).
- Нужны кэш на диск, паузы между запросами Steam и хранение токенов — это
  серверная работа.

Поэтому проект устроен так:

- **`spread-arbitrage.html`** — интерфейс. Открывается сам по себе и **работает
  в демо-режиме** на встроенных мок-данных (никакого сервера не надо).
- **`server.js`** — бэкенд на **чистом Node.js без единой зависимости** (только
  встроенные модули, `npm install` не требуется). Он проксирует и кэширует
  LIS / Steam, считает спред в валюте пользователя и отдаёт готовый результат.

Когда сервер запущен, страница **сама его находит** и переключается в
**live-режим** с реальными ценами (бейдж в шапке меняется `demo → live`). Если
сервера нет — остаётся демо.

## Быстрый старт

### Вариант 1. Демо (без сервера)
Просто открой `spread-arbitrage.html` в браузере. Всё работает на демо-данных —
видно интерфейс, расчёт спреда, сортировку, режим «чистыми», выбор валюты.

### Вариант 2. Live (реальные цены)
Нужен только Node.js (≥16), зависимостей нет.

```bash
node server.js            # поднимет сервер на http://localhost:8787
# открой http://localhost:8787 в браузере
```

Без настройки источников сервер всё равно отдаёт демо-данные (и честно пишет об
этом предупреждением). Чтобы пошли **реальные** цены — подключи источник ниже.

## Подключение реальных цен

Самый быстрый путь — **API-токен LIS-Skins**. Тогда прайс **не выгружается
целиком**: на каждый поиск сервер точечно запрашивает `market/search` только по
нужному ценовому коридору (`price_from`/`price_to`). Это на порядок быстрее
старой выгрузки bulk (~640 МБ).

```bash
# LIS через API-токен — точечный market/search (рекомендуется, быстро)
LIS_API_TOKEN=your_token \
MODE=live node server.js

# Запасной путь: локальный JSON-файл прайс-листа (без токена)
LIS_PRICES_FILE=./data/lis_prices.json \
MODE=live node server.js

# Запасной путь: публичный bulk (640+ МБ, медленно при первой загрузке)
MODE=live node server.js
```

**Валюты без внешнего FX.** Цены в регионах Steam независимы, поэтому Steam
запрашивается **сразу в валюте поиска** (`priceoverview&currency=<код>`:
USD=1, EUR=3, RUB=5, UAH=18, KZT=37). LIS работает в USD — его цена
переводится в валюту по курсу, который берётся **из самого Steam**: эталонный
ликвидный скин (`FX_REF_ITEM`) запрашивается в USD и в нужной валюте,
отношение цен = курс (кэш `FX_TTL`, фолбэк — встроенные константы). Никакого
стороннего сервиса курсов больше нет.

> Спред считается в валюте показа: Steam — нативно, LIS — `USD × курс`.

### Все переменные окружения

| Переменная | Назначение | По умолчанию |
|---|---|---|
| `PORT` | порт сервера | `8787` |
| `MODE` | `demo` / `live` / `auto` | `auto` |
| `LIS_API_TOKEN` | токен LIS API → быстрый `market/search` (приоритет №1) | — |
| `LIS_API_BASE` | база LIS API | `https://api.lis-skins.com/v1` |
| `LIS_GAME` | игра для `market/search` | `csgo` |
| `LIS_SEARCH_PAGES` | сколько страниц `market/search` листать на поиск | `6` |
| `LIS_SEARCH_MAX` | потолок уникальных скинов из `market/search` | `120` |
| `LIS_PRICES_FILE` | локальный прайс LIS (запасной путь, без токена) | — |
| `LIS_BULK_URL` | bulk-JSON по ссылке (запасной путь, без токена) | публичный bulk |
| `FX_REF_ITEM` | эталонный скин для курса USD→валюта из Steam | `AK-47 \| Redline (Field-Tested)` |
| `FX_TTL` | TTL кэша курса, сек | `21600` (6 ч) |
| `STEAM_DELAY_MS` | пауза между запросами Steam | `3500` (~17/мин) |
| `STEAM_MAX_LOOKUPS` | потолок «свежих» запросов Steam на один поиск | `40` |
| `STEAM_TTL` | TTL кэша цен Steam, сек | `21600` (6 ч) |
| `PRICES_TTL` | TTL кэша прайса LIS (только запасной путь) | `1800` (30 мин) |
| `STEAM_FEE` | комиссия Steam для режима «чистыми» | `0.15` |
| `CACHE_DIR` | папка дискового кэша | `.cache` |
| `LOG_LEVEL` | `debug` для подробного логирования | — |

> С токеном LIS прайс **не выгружается целиком** — поиск точечный по коридору.
> Без токена остаётся запасной путь (файл/bulk): bulk ~640 МБ агрегируется до
> ~19K скинов и кэшируется; первая загрузка долгая, повторные — из кэша.

## Как это работает (по шагам)

1. **Коридор цены.** Вокруг введённой суммы строится диапазон ±N% (в USD;
   сумма пользователя переводится в USD по курсу из Steam).

2. **Поиск LIS по цене.** С токеном сервер дёргает `market/search` с
   `price_from`/`price_to` и забирает только скины из коридора (cursor-пагинация,
   агрегирует до минимальной цены за скин). Без токена — фильтрует предзагруженный
   прайс. Никакой выгрузки 640 МБ на каждый запуск.

3. **Сверка со Steam — в валюте поиска.** Для кандидатов запрашивается цена Steam
   Market (минимальная, иначе медианная) сразу в нужной валюте
   (`currency=<код>`). Цены в регионах Steam независимы — внешний курс не нужен.
   С паузами, кэшем и повторами по 429.

4. **Спред и рейтинг.** LIS (USD) переводится в валюту по курсу из Steam.
   Спред = выплата LIS − цена Steam (в режиме «чистыми» ещё −~15%). Кандидаты
   сортируются по спреду; сверху — лучшая сделка со ссылками на обе площадки.

## CLI

```bash
node server.js search 8000                      # поиск по сумме (рубли, ±5%)
node server.js search 8000 --currency KZT --corridor 8 --net
node server.js search 100  --currency USD
node server.js fx                               # курсы USD→валюта (из Steam)
node server.js steam "AK-47 | Asiimov (Field-Tested)" --currency RUB  # цена скина
node server.js diag                             # состояние источника LIS
node server.js serve --port 9000                # поднять сервер на другом порту
```

## HTTP API

Все ответы — JSON, с заголовком `Access-Control-Allow-Origin: *`.

| Endpoint | Описание |
|---|---|
| `GET /api/health` | статус, режим, источник LIS, uptime |
| `GET /api/search?amount=&currency=&corridor=&net=&mode=` | главный расчёт (в валюте) |
| `GET /api/fx` | курсы USD→валюта (из Steam, кэш/константы) |
| `GET /api/steam?name=<market_hash_name>&currency=RUB` | цена скина в Steam (нативно) |
| `GET /api/prices?limit=N` | срез прайса LIS (запасной путь без токена) |

## Ограничения и нюансы

- **Лимиты Steam.** ~20 запросов/мин. Проверяются только кандидаты из коридора,
  с паузами и кэшем. При 429 — повтор с увеличенной паузой.
- **Несовпадение имён.** Имя на площадке должно совпадать с `market_hash_name` в
  Steam, включая ★ и StatTrak™. Несовпавшие показываются отдельным списком.
- **Точность.** Цены на всех площадках меняются — результат актуален на момент
  запуска.
